# 🧊 WhatToEat（吃什么）

> 基于冰箱食材管理 + 智能菜谱推荐 的微信小程序  
> 第一阶段目标：先实现微信小程序版本，后续扩展到 App（RN）

---

## ⚙️ 技术架构

| 模块 | 技术栈 | 说明 |
|------|----------|------|
| 前端 | **Taro + React + TypeScript** | 主体小程序框架，方便未来扩展到 RN / Web |
| 状态管理 | **Zustand**（推荐） | 简洁轻量、适合小程序架构 |
| UI组件 | **Taro UI / NutUI / 自定义组件** | 小程序端 UI 支持 |
| 图片识别 | **腾讯云 / 百度AI 图像识别 API** | 拍照识别食材类别 |
| 后端（可选阶段） | **Supabase / Firebase / Node + Express + MongoDB** | 云端同步、账户管理 |
| 存储 | **微信小程序 Storage + 云开发数据库** | 本地离线 + 云端备份 |
| CI/CD | **GitHub Actions + Taro 构建 + 微信小程序上传** | 自动打包、预览与上传（后期阶段） |

---

## 📱 第一阶段功能模块（微信小程序版）

### 1️⃣ 食材管理模块
**核心目标：管理冰箱食材**
- ✅ 拍照识别食材（接入 AI 识别 API）     
- ✅ 手动录入（输入名称、分类、保质期）
- ✅ 编辑 / 删除食材
- ✅ 根据保质期排序（快过期 → 置顶提醒）
- ✅ 支持分类视图（蔬菜 / 肉类 / 饮品 / 其他）
- ✅ 支持「冰箱区块」管理（冷藏区、冷冻区）

---

### 2️⃣ 智能推荐模块
**核心目标：根据冰箱内容推荐吃什么**
- ✅ 根据食材组合推荐菜谱（AI 或内置算法）
- ✅ 支持筛选模式（如：减脂 / 家常 / 快手）
- ✅ 根据保质期优先推荐快过期食材做菜
- ✅ “心情推荐”：随机 or 精选菜谱模式

---

### 3️⃣ 食材详情页
- ✅ 展示名称、分类、保质期
- ✅ 自动计算剩余天数
- ✅ 显示关联菜谱推荐
- ✅ 支持延长保质期或标记已食用

---

### 4️⃣ 通知提醒模块（后期）
- 🔔 每日提醒即将过期食材
- 🔔 每周推荐新的菜谱清单
- 🔔 可手动开启/关闭通知推送

---

### 5️⃣ 用户数据模块（后期）
- 👤 本地账号登录 / 微信授权登录
- ☁️ 云端同步数据
- 📦 多设备同步（配合未来 App）
